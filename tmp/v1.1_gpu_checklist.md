# ICARION v1.1 GPU Integration Checklist

## Scope for v1.1
- Deliver a usable GPU backend for supported paths (integration, fields, collisions, space charge) with documented limits.
- Keep CPU as baseline; GPU offload is optional and must fail safe to CPU.
- No promise of bitwise parity; statistical parity with tolerance is acceptable (state clearly).

## Work Items

### 1) Inventory & Cleanup
- [ ] Consolidate IonStateGPU definition (single header, SoA); remove unused AoS header under core/types.
- [ ] Decide fate of TransferDomainData/GPUParams (remove or move under core/gpu; wire or drop).
- [ ] Clarify GPU enum/location (InstrumentEnums/InstrumentGPU) and ensure SSOT with CPU enums.

### 2) Integrators (RK4/RK45/Boris)
- [ ] Stabilize GPU RK4/RK45/Boris kernels; ensure they compile and run on target architectures.
- [ ] Wire field-provider support: CPU → GPU texture upload for imported HDF5 field maps; graceful fallback if invalid.
- [ ] Add validation tests vs. CPU integrators (position/velocity tolerance-based, not bitwise), including with fields.
- [ ] Document that GPU integrators are optional; default remains CPU.

### 3) Field Providers
- [ ] End-to-end path: HDF5 field map → FieldArrayGPU textures → GPU integrators.
- [ ] Handle missing/invalid maps: explicit fallback to zero fields; log warning.
- [ ] Confirm analytical fields are handled consistently on GPU (or explicitly state unsupported).

### 4) Collisions (HSS/EHSS)
- [ ] Confirm GPU HSS/EHSS kernels are functional; map species_id → geometry index (no stubbed single-species).
- [ ] Add CPU vs. GPU statistical parity tests for HSS (velocity distributions) and EHSS (if geometry available).
- [ ] If EHSS is not viable on GPU, disable and document; keep CPU as reference.

### 5) Space Charge
- [ ] Decide on GPU P3M: integrate into Force path or disable; if integrated, add CPU vs. GPU comparison (small N vs. O(N²)).
- [ ] Document geometry limits (rectangular grid only; no cylindrical/Orbitrap) and boundary behavior.
- [ ] Ensure CPU direct O(N²) and box-grid Poisson remain available as baseline.

### 6) Boundaries & Domains
- [ ] GPU boundary checks aligned with CPU (cylindrical domains; Orbitrap handled on CPU). Document unsupported geometries.
- [ ] Domain ID handling consistent between CPU/GPU paths.

### 7) RNG & Reproducibility
- [ ] Define RNG policy: std::mt19937_64 on CPU; XORWOW on GPU. No per-ion persistent streams unless added explicitly.
- [ ] State reproducibility expectations: no bitwise parity; statistical equivalence within tolerance.

### 8) Tests & CI
- [ ] Add GPU ctests for integrators (RK4/RK45/Boris), field interpolation, HSS collisions, and space charge (if enabled).
- [ ] Set tolerance-based assertions (positions/velocities, distributions).
- [ ] Provide CPU fallback tests to ensure safe degradation when GPU unavailable.

### 9) Documentation
- [ ] Update README/docs with a CPU vs. GPU feature matrix (what’s supported, experimental, disabled by default).
- [ ] Note geometry limits, field map support, space charge limits, RNG differences, and fallback behavior.

## Out of Scope for v1.1 (future)
- Full FieldSolver integration on GPU (v1.2 target).
- Advanced space-charge models beyond box-grid/P3M (v1.2 target).
- Long-range collision precomputation (v1.3 target).
- Physics-driven geometry optimizer (v2.0 target).
